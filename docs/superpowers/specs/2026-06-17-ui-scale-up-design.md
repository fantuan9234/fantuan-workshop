# UI 尺寸放大方案设计

## 概述

将 Stardew Mod Studio 的界面字体和模块尺寸整体放大一档，改善可读性和视觉舒适度。项目当前广泛使用 `text-[10px]`、`text-xs`、`text-sm` 等较小字号，模块如侧边栏、面板等尺寸偏小。

## 范围

覆盖 `src/renderer/src/` 下所有组件/页面文件，主要包括：

1. 字体大小逐级升一档
2. 模块宽度/高度按 +15~20% 比例放大
3. 全局 CSS 基类字号调整
4. 部分间距/内边距同步放大

## 详细映射

### 字体大小映射

| 当前值 | 替换为 | 实际变化 |
|---|---|---|
| `text-[9px]` | `text-[11px]` | 9 → 11px |
| `text-[10px]` | `text-xs` | 10 → 12px |
| `text-[11px]` | `text-sm` | 11 → 14px |
| `text-xs` | `text-sm` | 12 → 14px |
| `text-sm` | `text-base` | 14 → 16px |
| `text-base` | `text-lg` | 16 → 18px |
| `text-lg` | `text-xl` | 18 → 20px |
| `text-xl` | `text-2xl` | 20 → 24px |
| `text-2xl` | `text-3xl` | 24 → 30px |

### 模块尺寸映射

| 当前值 | 替换为 | 位置说明 |
|---|---|---|
| `w-[56px]` | `w-[64px]` | 侧边栏折叠态宽度 |
| `w-[180px]` | `w-[220px]` | 侧边栏展开态宽度 |
| `w-[260px]` | `w-[300px]` | 右面板宽度 |
| `h-10` | `h-12` | 标题栏高度 |
| `w-[360px]` | `w-[420px]` | 编辑面板 (lg) |
| `w-[380px]` | `w-[440px]` | 地图预览面板 (lg) |
| `w-[400px]` | `w-[460px]` | 属性面板 (xl) |
| `w-[420px]` | `w-[480px]` | 地图预览面板 (xl) |
| `w-[280px]` | `w-[320px]` | 脚本预览面板 (lg) |
| `w-[320px]` | `w-[360px]` | 编辑器左侧面板 (lg) |
| `h-8` | `h-10` | 部分小型面板高度 |
| `min-w-[56px]` | `min-w-[64px]` | 侧边栏最小宽度 |
| `min-w-[180px]` | `min-w-[220px]` | 侧边栏展开最小宽度 |
| `min-w-[260px]` | `min-w-[300px]` | 右面板最小宽度 |

### 间距/内边距调整

| 当前值 | 替换为 | 说明 |
|---|---|---|
| `p-3` | `p-4` | 部分紧凑容器内边距 |
| `px-3` | `px-4` | 部分水平内边距 |
| `gap-2` | `gap-3` | 部分间隔 |

### 全局样式调整

- `index.css` 中 `.input` 基类的 `font-size: 13px` → `font-size: 15px`
- 对应的 `padding: 6px 10px` → `padding: 8px 12px`

## 实施方式

1. 编写 Node 脚本 `scripts/scale-up-ui.js` 执行批量精确替换
2. 替换覆盖：字体类、宽度/高度类、间距类、全局 CSS
3. 人工核验少数特殊情况（如图标尺寸、特定布局约束等不宜替换的场合）
4. 构建并验证界面效果

## 不做的事

- 不改变布局结构（flex/grid 结构、组件层级不变）
- 不改图标大小（`w-4`/`h-4` 等保持不动，图标另有视觉标准）
- 不改颜色或主题
- 不改动后端/主进程代码
